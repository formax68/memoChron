<objective>
Implement ENH-01: render a persistent "today" indicator on the calendar grid that stays visible even when today is also the selected day. The fix is CSS-only — `createDayElement` in `src/utils/viewRenderers.ts:236-253` already adds the `.today` class on every surface that renders the grid (sidebar `CalendarView` and embedded `EmbeddedCalendarView`), so a single CSS rule paints the indicator everywhere.

Per D-01, the indicator is an **inset accent ring** rendered via `box-shadow: inset 0 0 0 2px var(--interactive-accent)`. Inset shadows paint inside the cell border so the ring overlays the cell's background — including the `var(--interactive-accent)` background of `.selected`. This satisfies ROADMAP success criterion #1: "Today's cell shows a distinct ring or border even when a different day is selected — both highlights are visible simultaneously."

This addresses **ROADMAP Phase 4 success criterion #1**.

Purpose: smallest possible change to make today always visible on the grid. The existing `.today .memochron-day-header { color: var(--interactive-accent); font-weight: 700 }` rule (around `styles.css:161-164`) was a partial solution but disappears when today is selected (the header text color flips to `var(--text-on-accent)` via the `.selected .memochron-day-header` rule). The inset ring is independent of header text color and survives the selected state.

- Zero TypeScript changes — `createDayElement` already does the work

Concrete values for the new rule:
- Selector: `.memochron-day.today`
- Single declaration: `box-shadow: inset 0 0 0 2px var(--interactive-accent);`
- Rationale comment on the line above the selector: `/* ENH-01: persistent today indicator — inset ring survives the .selected accent background */`

Why `inset` matters: a non-inset shadow paints outside the cell border and would be clipped by the grid container (and overlap neighbor cells); the inset form paints inside the cell border, on top of whatever background the cell already has — including `var(--interactive-accent)` when the cell is also `.selected`. Per D-01, this is what makes today and selected visible simultaneously.

Why `var(--interactive-accent)` and not a hard-coded color: per D-03, the ring color is hard-coded to the same Obsidian theme variable as the existing `.selected` background and the `.today .memochron-day-header` color. No new color setting; no theme fork; light/dark themes follow automatically. Do **not** introduce hex literals.

Why 2px and not 1px or 3px: 2px is the recommended thickness per CONTEXT.md "Claude's Discretion" — readable on standard zoom levels and matches typical Obsidian theme border weights. Stay at 2px unless visual review explicitly requests otherwise.
