Three-bucket policy (D-09):
  - **Bucket 1 — `void` operator**: true fire-and-forget where caller doesn't care about errors and a global error wouldn't be actionable. Examples:
    - `void this.refreshEventsInBackground();` — refresh on layout change; failures retry on next tick
    - `void this.loadDailyNotesAsync();` — opportunistic prefetch
  - **Bucket 2 — `.catch(error => new Notice(errorMessage(error)))`**: user-visible failure surface. DOMINANT choice. Examples:
    - User-clicked event opens a note → failure must surface as Notice
    - Settings save chain → failure should notify
    - First-load fetch in a view's `onload`/`initialize` → failure should notify
  - **Bucket 3 — `await`**: when the surrounding context is already async and execution should sequence. RARE — most identified sites are inside callback signatures that don't accept `await`.

Default for ambiguous sites per D-09: `.catch(error => new Notice(errorMessage(error)))` — safer than silent `void`.
