<objective>
TD-04: Add a `private isDragging = false` flag to `CalendarView`. Set it true in `handleDragStart`, false in `handleDragEnd`. Override `View.onClose()` (signature `protected async onClose(): Promise<void>`) to remove the two `window` listeners (`mousemove`, `mouseup`) and clear the flag if a drag is in progress when the view tears down. Do NOT call `handleDragEnd` itself from `onClose` — that path triggers `saveSettings` -> `refreshCalendarView` against a torn-down view and races teardown.

Purpose: Closing the sidebar (or disabling the plugin) mid-drag must not leave detached `window` listeners that fire later into a destroyed view. This is the second half of the iOS rapid disable/enable crash fix; Plan 03 added `detachLeavesOfType` to deterministically fire `onClose`, and this plan implements the cleanup that runs there.

Output: Modified `src/views/CalendarView.ts` (one new field, two flag-mutations in existing handlers, one new method). Atomic commit per requirement.

        this.isDragging = false;

    - Do NOT add code outside the `if (this.isDragging)` guard. If a drag is NOT in progress, `handleDragEnd` will have already run normally and removed the listeners — adding unguarded `removeEventListener` calls is harmless but pointless.
