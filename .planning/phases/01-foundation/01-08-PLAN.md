Plan numbering: 01-08 closes WR-02 from `.planning/phases/01-foundation/01-REVIEW.md`.

WR-02: `parseAgendaCodeBlock` (`src/views/EmbeddedAgendaView.ts:412`) and `parseCalendarCodeBlock` (`src/views/EmbeddedCalendarView.ts:250`) both split each config line on `":"` with `line.split(":").map(s => s.trim())`, then destructure `[key, value]`. When a value itself contains a colon — such as `title: My Meeting: Q2 Review` or `date: 2026-05-09T10:30` — the third-and-beyond segments are silently discarded. The most affected parameter is `title` (any title with a colon is truncated). The `date` parameter loses the time portion when it includes an ISO datetime with a colon. Per REVIEW.md these are the two known affected parameters; both parsers' switch statements accept multiple keys (`title`, `date`, `month`, `year`, `days`, `showpast`, `showdailynote`, `showdots`, `calendars`) — `title` and `date` are the practically affected ones, but the fix applies to every key/value pair.

Fix per REVIEW.md: replace the plain `split(":")` with `indexOf(":")` + `substring`. Both parsers share the same fix pattern; one plan covers both files.

Purpose: Code-block configurations with colons in values render correctly. After this plan, an agenda block like

```
date: 2026-05-09T10:30
title: My Meeting: Q2 Review
```

parses `date` as `2026-05-09T10:30` (not `2026-05-09T10`) and `title` as `My Meeting: Q2 Review` (not `My Meeting`).

Output: Two files modified, one parser per file. One atomic commit.

```typescript
export function parseAgendaCodeBlock(source: string): AgendaCodeBlockParams {
  const params: AgendaCodeBlockParams = {};
  const lines = source.trim().split("\n");

  for (const line of lines) {
    const [key, value] = line.split(":").map((s) => s.trim());

    // Skip lines without colons or empty keys
    if (!key || value === undefined) {
      continue;
    }

    switch (key.toLowerCase()) {
      case "date": params.date = value; break;
      case "days": params.days = parseInt(value); if (isNaN(params.days)) params.days = 1; break;
      case "showpast": case "show-past": params.showPast = value.toLowerCase() === "true"; break;
      case "showdailynote": case "show-daily-note": params.showDailyNote = value.toLowerCase() === "true"; break;
      case "title": params.title = value; break;
      case "calendars": params.calendars = value.split(",").map((s) => s.trim()).filter((s) => s.length > 0); break;
    }
  }

  return params;
}
```
